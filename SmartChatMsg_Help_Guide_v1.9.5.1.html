SmartChatMsg Help Guide (v1.9.5.1)

SmartChatMsg is an Elder Scrolls Online addon designed to make repeated chat posting faster and easier. It lets you define your own slash commands, save multiple message variants under each command, organize them by guild, and automatically populate the correct message into the correct chat channel.

It was built for things like guild recruitment ads, event reminders, trial announcements, officer notices, and other recurring messages that you want to reuse without retyping them over and over.

Important: SmartChatMsg fills the chat input for you. It does not press Enter or silently send the message. This gives you a chance to review the text before posting.

What SmartChatMsg Does

Custom Slash Commands

Create commands such as /recruit, /welcome, /event, or any other short name you want.

Guild-Based Message Sets

The same command can have different saved messages for different guilds.

Saved Output Channel

Each command and guild combination remembers whether it should populate to Zone, Guild, or Officer chat.

Automation Options

You can use commands on demand, repeat them, trigger them on zone changes, or mark them to run at startup.

Built-In Commands

  • /scm opens the SmartChatMsg settings panel.
  • /scmdebug toggles debug logging. You can also use /scmdebug on, /scmdebug off, and /scmdebug status.
  • /scm status toggles the status panel on or off.

Global Settings

Default Guild

This lets you choose which guild slot should be assumed when you run a command without providing a guild number.

Example: if your default guild is slot 3, then running /recruit behaves the same as running /recruit 3.

Auto-Remove Pending Chat after (secs)

This is the global timeout for populated chat text.

  • If SmartChatMsg fills text into chat and you do not send it, the addon waits for this timeout.
  • When the timeout is reached, SmartChatMsg restores your previous chat channel.
  • It also clears the pending chat buffer.
  • The minimum valid value is 30 seconds.
  • The default is 60 seconds.

Create / Edit / Delete Commands

Think of a command as the short slash command you type to trigger a saved message set. Examples might be recruit, welcome, trial, or greetings.

  1. Open settings with /scm.
  2. Go to the command area.
  3. Select a command from the dropdown to edit it, or click + to create a new one.
  4. Enter the command name and save it.
Command naming note: command names are sanitized before registration as slash commands. In practice, keep names simple and use letters, numbers, or underscores for the cleanest results.

Create / Edit / Delete Messages

This is where most of the setup work happens.

  1. Select the command you want to configure.
  2. Select the guild that command should apply to.
  3. Choose the output channel.
  4. Set any behavior options you want.
  5. Add one or more saved messages.
  6. Save your changes.

Once command and guild are selected, SmartChatMsg stores behavior settings at the Command + Guild level. That means one command can behave differently for different guilds.

Output Channel

Output Channel determines where the text will be populated.

  • Zone
  • Guild (/g#)
  • Officer (/o#)

When SmartChatMsg populates a message, it temporarily switches chat to the configured destination, fills in the selected message, and restores your prior chat channel after you send it. If you do not send it before timeout, the addon restores the previous channel and clears the pending text.

Run At

The Run At dropdown stores how the command is intended to be started for that command and guild combination.

  • On Demand — you trigger it manually by typing the slash command.
  • Startup — the command is added to a startup queue after login or reload and is processed automatically.
  • Scheduled — reserved for future scheduling behavior.

How Startup Works

When the addon finishes loading, it gathers any command+guild entries marked for Startup and builds a queue. The queue is processed from top to bottom. Each queued item is handled the same way as if you had typed that slash command yourself for the correct guild context.

  • If the populated text is actually sent, that queue item is considered complete and removed.
  • If the text times out and is cleared instead of being sent, that item can be retried later according to the queue logic.
  • The queue only processes one item at a time.

Repeat Every (mins)

Repeat Every creates a repeating cycle for the selected command and guild.

  1. Run the slash command once.
  2. SmartChatMsg records the last-used state for that command and guild.
  3. After the configured number of minutes, it repopulates a message into chat.
  4. Once that populated message is confirmed as sent, the next repeat cycle is scheduled.
Important: Repeat Every and Auto Populate Chat on Zone are mutually exclusive. Setting a valid Repeat value turns off Auto Populate Chat on Zone for that same command and guild. Turning on Auto Populate Chat on Zone clears Repeat Every for that same command and guild.

Retry Delay (mins)

Retry Delay only matters when Repeat Every is active.

  • If a repeated message is populated but not actually sent, SmartChatMsg can try again after the retry delay.
  • If Retry Delay is 0, blank, or invalid, SmartChatMsg skips retry and resumes the normal repeat cycle after the populate attempt times out.
  • Retry Delay is automatically capped so it cannot exceed Repeat Every.

Auto Populate Chat on Zone

Auto Populate Chat on Zone is the alternative automation mode. It is useful when you want SmartChatMsg to prepare a message after zoning into a valid area, provided enough time has passed since the last populate for that zone.

How it behaves

  1. Run the slash command once to activate it for the selected command and guild.
  2. Travel into a valid parent zone, or log in or reload while already in one.
  3. SmartChatMsg evaluates whether that zone is eligible and whether the cooldown has expired.
  4. If eligible, it populates the message into chat for you to send.
  • Only one active auto-populate command can run at a time.
  • The first player activation after login is ignored.
  • Zone checks also happen on startup.
  • If the current zone is still in cooldown on startup, SmartChatMsg shows a cooldown alert instead of populating immediately.
  • It only fires for eligible parent zones.
  • Running the command again, or using off, turns it off.

Cooldown (mins)

Cooldown applies to Auto Populate Chat on Zone.

  • Cooldown is tracked per zone.
  • This prevents the same command from repopulating too quickly for the same zone.
  • Default value is 60 minutes.
  • Invalid, blank, or non-positive values normalize back to 60.

Open Status Panel on Run

This checkbox tells SmartChatMsg to open the status panel automatically when the command is run in a context where automation status is meaningful.

The status panel is especially useful for zone-based automation because it shows whether auto-populate is active and displays cooldown information for tracked zones.

Notify Sound

Because it is easy to miss when text has been populated into chat, you can choose a sound to play as a notification.

  • Notify Sound is stored per Command + Guild.
  • The default value is DUEL_START.
  • None disables sound completely.
  • The sound plays whenever SmartChatMsg successfully populates text into chat, including manual slash command starts, Repeat Every, and Auto Populate Chat on Zone.
  • The Preview button lets you test the selected sound immediately from settings.

Matching Messages and Message Rotation

One of the biggest strengths of SmartChatMsg is that a command can have multiple saved messages for the same guild. Instead of always using the same text, the addon rotates through your entries using weighted selection.

Messages that have never been used are favored first, and messages used less recently or less often are more likely to be chosen than ones you just used. This helps your saved messages rotate naturally.

Example: if you have three recruitment ads saved under the same command and guild, SmartChatMsg will vary which one is selected instead of always choosing the same entry.

Editing and Deleting Messages

You can edit existing messages directly in the message list, add new ones, delete individual messages, or select multiple entries and delete them together.

Built-In Substitutions

Messages support wildcard substitutions that are replaced when the message is populated.

Token What it becomes
%time% morning, afternoon, or evening based on current time of day
%guild% The resolved guild name for the current command context
%zone% Your current zone name

These tokens are case-insensitive, so %TIME%, %Guild%, and %Zone% also work.

Slash Command Usage

If you create a command named recruit, SmartChatMsg registers /recruit.

Example What it does
/recruit Uses the default guild if one is configured.
/recruit 1 Targets guild slot 1.
/recruit g1 Also targets guild slot 1.
/recruit o1 Still targets guild slot 1. The parameter resolves the guild slot, not the final output channel.
/recruit off Turns off repeat and/or auto-populate for the default guild context.
/recruit 2 off Turns off repeat and/or auto-populate for guild slot 2.

No matter how a command is configured, you can still run it manually at any time. Adding off is the clear way to stop automation for that command and guild.

Status Panel

The status panel gives you a quick view of the current automation state. When Auto Populate Chat on Zone is active, it shows:

  • whether automation is active,
  • the command currently associated with the active auto-populate state,
  • the guild name,
  • the output channel,
  • the current zone,
  • tracked zone cooldown rows,
  • and a footer showing how many zones are being shown.

When no active auto-populate command exists, the panel shows Auto: Inactive and a footer indicating zero tracked zones.

Status panel example: this screenshot shows the SmartChatMsg status panel in its inactive state, including the header, close button, current status line, and tracked-zones footer.
Tip: You can close the status panel by clicking the X in the upper-right corner, or by using /scm status to toggle it.

Import / Export

SmartChatMsg includes import and export support so you can back up or share your setup.

Export includes

  • Commands
  • Messages
  • Saved output channels
  • Per-command/per-guild behavior settings
  • Default guild
  • Global revert timeout
  • Active auto-populate state

Import behavior

  • Import replaces existing SmartChatMsg settings.
  • Import requires confirmation before applying.
  • Success and error feedback are shown in game.

Typical Example Setup

  1. Open settings with /scm.
  2. Create a command named recruit.
  3. Select that command in the messages section.
  4. Select the guild it should use.
  5. Set Output Channel to Zone.
  6. Add several recruitment messages.
  7. Choose whether you want it to run only on demand, repeat, auto-populate on zone, or queue at startup.
  8. Use /recruit, /recruit 1, /recruit g1, or /recruit o1 as needed.

Important Limitations and Notes

  • SmartChatMsg cannot press Enter for you. It can only populate the chat buffer.
  • Only one Auto Populate Chat on Zone command can be active at a time.
  • Scheduled mode is present as a stored option but full scheduling behavior is not implemented yet.
  • Guild and Officer parameter forms such as g1 and o1 identify the guild slot. They do not override the saved output channel.

Debugging

If you need to diagnose why a command is not behaving as expected, use /scmdebug. This toggles SmartChatMsg debug logging.
