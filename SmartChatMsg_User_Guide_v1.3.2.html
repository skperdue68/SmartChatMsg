SmartChatMsg Detailed User Guide

Addon: SmartChatMsg
Version: 1.3.3
Game: The Elder Scrolls Online
Dependency: LibAddonMenu-2.0

SmartChatMsg lets you create custom slash commands that pull from saved message lists, choose a message for you, and place it into the correct chat channel. It is designed for guild recruitment, scheduled announcements, repeated informational posts, and other messages you want to reuse without manually typing them every time.

1) Overview

SmartChatMsg is built around three core ideas:

  • Commands such as /recruit or /trial.
  • Guild-aware message groups so the same command can behave differently depending on the selected guild.
  • Saved output channels so the addon knows whether a message should go to Zone, Guild, or Officer chat.

When you run a SmartChatMsg slash command, the addon looks up the matching command, resolves the guild context, retrieves a saved message, applies substitutions such as %time%, %guild%, and %zone%, and then places the final text into the appropriate chat buffer.

2) Quick Start

  1. Open the addon settings in ESO.
  2. Create a new command name, for example recruit.
  3. Go to the Messages section.
  4. Select the command.
  5. Select a guild, or choose Any (Guild/Officer only).
  6. Choose the output channel.
  7. Add one or more saved messages.
  8. Use the slash command in chat, such as /recruit 1, /recruit g1, or /recruit o1.
Tip: Save several variants of the same message. SmartChatMsg can rotate among them so your posts do not feel repetitive.

3) How Commands Work

Each command name becomes a slash command. If you create a command named recruit, you can use it as /recruit.

Supported parameter formats

FormatMeaning
/command 1 to /command 5Use the guild in slot 1 through 5.
/command g1 to /command g5Force Guild chat for guild slot 1 through 5.
/command o1 to /command o5Force Officer chat for guild slot 1 through 5.
/commandUses the Default Guild if one is configured.
If no parameter is provided and no Default Guild is set, the slash command cannot resolve a guild target.

What happens when you run a command

  1. The addon identifies the command.
  2. It resolves the target guild from the parameter or Default Guild.
  3. It chooses the saved channel for that command and guild context, unless a channel override was provided by g# or o#.
  4. It retrieves a matching saved message.
  5. It applies substitutions to the retrieved text.
  6. It places the finished text into the chat input buffer.
  7. It restores your previous chat channel after the message is sent, or after the restore timeout.

4) Managing Messages

Messages are stored under a specific Command + Guild pairing. You can save a single message or multiple alternatives.

Suggested workflow

  1. Select the command.
  2. Select the guild context.
  3. Select the output channel.
  4. Add multiple message variants.
  5. Save and test the slash command.

Why save multiple variants

Saving multiple versions of the same announcement lets SmartChatMsg rotate messages rather than repeating the exact same line every time.

Example message set for one recruitment command:

Join %guild% this %time% for trials and events.
%guild% is recruiting active players in %zone%.
Good %time%! %guild% is looking for social and PvE members.

5) Guild Selection and the Any Option

Normal guild selection

Normally, messages are saved for one specific guild. If you save messages for guild slot 2, those messages are intended for that guild's name and saved behavior.

Any (Guild/Officer only)

The Any selection lets you save a generic guild-facing message group that can be reused with any actual guild.

This is useful when the same message wording should work for several guilds without maintaining separate copies.

BehaviorResult
Saved under Any + Guild chatMay be used for any guild when the command resolves to a real guild slot.
Saved under Any + Officer chatMay be used for any guild's officer channel.
Saved under Any + Zone chatNot allowed.
Messages saved under Any are restricted to Guild or Officer output. They are not valid for Zone chat.

6) Chat Channel Behavior

Supported saved output channels

  • Zone
  • Guild
  • Officer

Automatic channel restore

After SmartChatMsg fills the chat input, it watches for the outgoing message and restores your previous chat channel. If it does not detect the sent message, a fallback timeout restores the previous channel automatically.

This behavior helps prevent getting stuck in the wrong chat tab after using a command.

7) Message Tokens and Substitutions

SmartChatMsg supports built-in tokens that are replaced in the retrieved message before it is sent to the chat buffer.

TokenDescriptionExample output
%time%Replaced with morning, afternoon, or evening.Good evening
%guild%Replaced with the resolved guild name for the current command context.The Silver Wolves
%zone%Replaced with the player's current zone name.Stormhaven

Case-insensitive tokens

Tokens are case-insensitive. These all work:

%time%
%Time%
%TIME%
%guild%
%GUILD%
%zone%
%Zone%

Inline replacements are supported

Tokens can appear anywhere in the message:

Good %time%, %guild%!
Recruiting now in %zone%.
Welcome to %GUILD% this %TIME%.

Important note about %guild%

%guild% is based on the resolved guild context for the command. It does not require you to currently be in Guild or Officer chat in order to substitute the guild name.

8) Reminders and Auto Populate

Reminder Minutes

You can set reminder minutes for the currently selected Command + Guild. After a command is used, SmartChatMsg can remind you if that command has not been used again within the configured interval.

Auto Populate On Zone

You can also enable automatic population when changing zones. This setting is stored per Command + Guild.

Auto Populate only triggers in parent zones. This helps reduce unwanted triggers while moving through sub-zones or interiors.

Reminder memory

Reminder tracking also remembers the last parameter used, such as the selected guild number, so reminder messages can reflect the most recent command context.

9) Import and Export

SmartChatMsg includes a full settings backup system.

Export

  • Creates a text backup string.
  • Displays a success message.
  • Places the export data in the text box.
  • Includes a Copy button to make copying easier.

Import

  • Accepts a previously exported SmartChatMsg backup string.
  • Shows a confirmation prompt before replacing existing settings.
  • Clears the text box after a successful import.
  • Displays a success message.
Import replaces your existing SmartChatMsg settings. Export first if you want a backup before importing a different configuration.

10) Example Setups

Example A: Zone recruitment for one guild

  1. Create command recruit.
  2. Select guild slot 1 in the Messages section.
  3. Set channel to Zone.
  4. Add several messages using %guild% and %time%.
  5. Use /recruit 1.

Example B: Generic officer notice for any guild

  1. Create command officermsg.
  2. Select Any (Guild/Officer only).
  3. Set channel to Officer.
  4. Add messages such as Good %time%, officers of %guild%.
  5. Use /officermsg o2 or another officer parameter for the intended guild slot.

Example C: Default Guild shortcut

  1. Set a Default Guild.
  2. Create command trial.
  3. Save messages for that guild.
  4. Use /trial without parameters.

11) Troubleshooting

The command says it needs 1-5, g1-g5, or o1-o5

The command was used without a valid parameter and no Default Guild is set.

No message appears

Verify that you have:

  • Selected the correct command
  • Saved messages for the intended guild context
  • Saved a valid output channel

Any does not work with Zone

This is expected. The Any option is restricted to Guild and Officer output.

%guild% or %zone% was not replaced

Check that the token is typed with percent signs, such as %guild% or %zone%. Token matching is case-insensitive, but the percent-based token shape still matters.

My channel did not switch back

The addon includes both direct message detection and a fallback timeout restore. If restore did not occur immediately, wait for the timeout. If the issue persists, test with fewer addon conflicts.

12) Best Practices

  • Keep command names short and easy to remember.
  • Use several message variants for the same purpose.
  • Use %time%, %guild%, and %zone% to keep messages feeling fresh.
  • Use Any only for true cross-guild Guild/Officer messages.
  • Export your settings before major changes.
