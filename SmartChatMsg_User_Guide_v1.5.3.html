SmartChatMsg User Guide (v1.5.3)

SmartChatMsg is an Elder Scrolls Online addon for players who regularly post reusable chat messages such as guild recruitment ads, trial announcements, officer notices, and other repeated chat content.

The addon lets you create your own slash commands, store multiple message variants for each command, organize those messages by guild, and populate the correct message into chat with the proper channel selected.

Important: SmartChatMsg fills the message into the chat input. It does not silently send the message for you. This gives you a chance to review the text before posting it.

What Is New in Version 1.5.3

  • Added Populate Sound at the Command + Guild level.
  • Added a Preview button for testing the selected sound.
  • Limited the sound dropdown to a curated set of useful ESO sounds.
  • Included DUEL_START by default and None for silent operation.
  • Populate sound now plays when a message is automatically populated by Auto Populate Chat on Zone or Repeat Every.

Main Design of the Addon

Commands

You create your own custom slash commands such as /recruit, /trial, or /officernotice.

Guild-Specific Messages

Each command can have different saved messages for different guilds.

Saved Output Channel

Each command and guild combination remembers whether it should populate to Zone, Guild, or Officer chat.

Automation Options

You can configure repeating behavior or zone-based auto population for each command and guild combination.

Available Included Commands

  • /scm opens the SmartChatMsg settings panel.
  • /scmdebug toggles debug logging. You can also use /scmdebug on, /scmdebug off, and /scmdebug status.

How to Set Up a Command

  1. Open settings with /scm.
  2. Add a command name.
  3. Go to the Messages section.
  4. Select the command you want to configure.
  5. Select the guild the messages belong to.
  6. Choose the output channel.
  7. Add one or more saved messages.
  8. Optionally configure Repeat, Retry, Auto Populate on Zone, Cooldown, and Populate Sound.
Command naming note: command names are sanitized before registration as slash commands. In practice, keep command names simple and use letters, numbers, or underscores for the cleanest results.

How Slash Commands Work

If you create a command named recruit, SmartChatMsg registers /recruit.

Example What it does
/recruit Uses the default guild if one is configured.
/recruit 1 Targets guild slot 1.
/recruit g1 Also targets guild slot 1.
/recruit o1 Still targets guild slot 1. The parameter resolves the guild slot, not the final output channel.
/recruit off Turns off repeat and/or auto populate for the default guild context.
/recruit 2 off Turns off repeat and/or auto populate for guild slot 2.

Using the command normally starts or restarts the selected behavior for that command and guild. Adding off is the clear way to stop the automation for that specific command and guild.

Default Guild

The Default Guild setting is global. It is used when you run a slash command without specifying a guild slot.

Example:

  • If your default guild is guild slot 3, then /recruit behaves like running the command for guild 3.
  • If you want to override that, use /recruit 1, /recruit 2, and so on.

Messages and Message Rotation

Each command can contain multiple saved messages for the same guild. This is useful for rotation so your repeated messages do not always look identical.

When SmartChatMsg needs to populate a message, it uses a weighted selection system that favors entries that were used less recently and less often. That means newer or less-used entries are more likely to be selected than one that was just used several times.

Built-In Substitutions

  • %time% becomes morning, afternoon, or evening.
  • %guild% becomes the resolved guild name.
  • %zone% becomes your current zone name.

These tokens are case-insensitive, so %TIME%, %Guild%, and %Zone% work too.

Output Channel

Output Channel is stored per Command + Guild.

Available choices are:

  • Zone
  • Guild (/g#)
  • Officer (/o#)

When SmartChatMsg populates a message, it switches the chat input to the configured destination, fills in the selected message, and later restores your previous chat channel after the message is sent.

Automatic Chat Restore and Pending Chat Timeout

The global Auto-Remove Pending Chat timeout controls how long SmartChatMsg waits before restoring your original channel if the populated message is not sent.

  • This timeout applies to all commands.
  • If the timeout is reached, SmartChatMsg restores the previous chat channel.
  • It also clears the pending chat buffer.
  • The minimum valid value is 30 seconds.
  • The default is 60 seconds.

Behavior Settings

These settings are stored per Command + Guild.

  • Repeat Every (mins)
  • Retry Delay (mins)
  • Auto Populate Chat on Zone
  • Cooldown (mins)
  • Populate Sound

Settings automatically save when changed.

Repeat Every (mins)

Repeat Every creates a repeat cycle for the selected command and guild.

How it behaves

  1. Run the slash command once.
  2. SmartChatMsg records the command state for that guild.
  3. After the configured number of minutes, it repopulates a message into chat.
  4. Once that message is confirmed as sent, the next repeat cycle is scheduled.
Important: Repeat Every and Auto Populate Chat on Zone are mutually exclusive in the settings flow. Setting a valid Repeat value turns off Auto Populate on Zone for that same command and guild. Turning on Auto Populate on Zone clears Repeat Every for that same command and guild.

Retry Delay (mins)

Retry Delay only matters when Repeat Every is active.

  • If a repeated message is populated but not sent, SmartChatMsg can try again after the retry delay.
  • If Retry Delay is 0, blank, or invalid, SmartChatMsg skips retry and resumes the normal repeat cycle after the populate attempt times out.
  • If Retry Delay is higher than Repeat Every, it is automatically reduced to match the repeat interval.

Auto Populate Chat on Zone

Auto Populate Chat on Zone is the alternative automation mode.

How it behaves

  1. Run the slash command once to activate it.
  2. Travel into a different parent zone.
  3. SmartChatMsg can populate a message for that zone.
  4. You can then send the message from chat input.
  • Only one active auto-populate command can run at a time.
  • The first player activation after login is ignored.
  • It only reacts to actual zone changes.
  • It only fires for parent zones.
  • Running the command again, or using off, turns it off.

Cooldown (mins)

Cooldown applies to Auto Populate Chat on Zone.

  • Cooldown is tracked per zone.
  • This prevents the same command from repopulating too quickly for the same zone.
  • Default value is 60 minutes.
  • Invalid, blank, or non-positive values normalize back to 60.

Populate Sound

  • Populate Sound is stored per Command + Guild.
  • The default value is DUEL_START.
  • The dropdown includes None and a curated set of useful ESO sounds.
  • When set to None, no populate sound is played.
  • The sound plays when text is automatically populated into chat by Auto Populate Chat on Zone or Repeat.
  • The Preview button plays the currently selected sound immediately.

Import and Export

SmartChatMsg supports exporting and importing addon settings.

Export includes

  • Commands
  • Messages
  • Saved output channels
  • Per-command/per-guild behavior settings
  • Default guild
  • Global revert timeout
  • Active auto-populate state

Import behavior

  • Import replaces existing SmartChatMsg settings.
  • A confirmation dialog is shown before import is applied.
  • Success and error feedback are shown in game.

Recommended Example Workflow

  1. Open settings with /scm.
  2. Create a command such as recruit.
  3. Select that command in the Messages section.
  4. Select your target guild.
  5. Choose Zone as the output channel.
  6. Add several recruitment messages.
  7. Decide whether you want timed reposting with Repeat Every or travel-based prompting with Auto Populate Chat on Zone.
  8. Choose a Populate Sound if desired.
  9. Run /recruit or /recruit 1 in chat.

Tips

  • Save multiple messages per command and guild to improve rotation variety.
  • Use %guild% and %zone% to reduce how many message variants you need to maintain.
  • Use Repeat for scheduled reposting.
  • Use Auto Populate on Zone for messages tied to travel or arrival.
  • Use off when you want a clear and explicit stop command for automation.
